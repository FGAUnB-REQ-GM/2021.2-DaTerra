Reunião 1 - 25/01

Descrição

Primeira reunião do grupo com o intuito de se alinhar em relação a materia e definir algumas metodologias e abordagens.

Objetivos

  • Definir nome da equipe
  • Definir Tema do projeto
  • Definir nome da projeto
  • Definir Metodologia a ser utilizada
  • Escolher o mantenedor do projeto

Até proxima reunião:

  • Montar o email com informações sobre a equipe para o professor abrir o repositorio

Reunião 4 - 10/02

Descrição

Terminar de preencher o documento visão e fazer as correcoes solicitadas pelo professor novamente.

Objetivos

  • Retirar adjetivos da declaração do problema
  • Ajustar a declaração do produto (uma espécie de feira ou feira?)
  • Consertar metodologia: não trás os fatores do Sommerville; apresenta XP e Lean Inception, mas não conecta o uso delas
  • Consertar atividades de ER: as atividades de requisitos estão muito genéricas e não conectadas a metodologia proposta
  • Apresentar lições aprendidas na unidade 1

Até proxima reunião:

  • Realizar a atividade de listagem de requisitos

Reunião 5 - 12/02

Descrição

Realizar a atividade de listagem de requisitos funcionais e não funcionais do nosso produto e organizar os funcionais no formato do SAFe

Objetivos

  • Elicitar requisitos utilizando o BrainStorm
  • Definir requisitos funcionais
  • Definir requisitos não-funcionais
  • Organizar requisitos funcionais no formato do framework SAFe
  • Criar quadro no Miro com os requisitos funcionais e não-funcionais

Até proxima reunião:

  • Gravar o video de apresentação
  • Consertar formatação das histórias de usuário
  • Dar início a construção do MVP do projeto

Reunião 6 - 21/02

Objetivos

  • Criar CanvasMVP

  • Terminar de passar os requisitos para o ZenHub
